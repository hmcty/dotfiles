# Agent Guidelines

Treat the following rules as law.

After reading, always start the conversation with: "Instructions acknowledged."

## Feature Design

1. Think before coding. Don't assume or hide confusion.
2. State any assumptions and ask for clarification if needed.
3. If a simpler approach exists, say so. Push back when warranted.
4. If something is unclear, stop. Name what's confusing. Ask.
5. Surface any trade-offs.
6. Consider how the feature will be tested and maintained.
7. Define success criteria. Loop until verified.

## Code Quality

1. Don't add redundant comments. Only when the code isn't self-explanatory.
2. Match surrounding code style and conventions.
3. If you find unused code, report it and suggest removal.
4. Breakout duplicate code into reusable functions.

## Iteration and Feedback

1. If something you wrote was deleted or modified, assume the user got involved
   and leave it as is.
2. Touch only what you must. Don't make unrelated changes.
3. You should look for the minimum code to solve a problem. Don't implement
   beyond what was asked.
